## 背景

平台组把 Harness Hub 作为组织级资产中心，结构为：

```
harness-hub/
├── packages/          # Skill、模版、Rubric
├── bundles/           # 拓扑 Bundle（api-service、frontend-2c …）
├── blueprints/        # 交付蓝图（组合 profile + hub_deps）
└── evals/             # golden-repo 验收集
```

某新立项的 **会员中心 API**（member-api）由后端负责人 **周工** 接入。平台组要求：**不得**从各仓库复制粘贴 Skill，必须从 Hub 消费；本地定制须走 `overrides` + `hub sync --apply` 三方合并。

角色：**赵老师**（平台组 Hub 维护者）、**周工**（member-api 负责人）。

## 操作步骤

### 1. 平台组：种子 Hub 与资产检索

```console
$ hx hub seed ./harness-hub
Seeded ./harness-hub with 8 package(s):
  api-conventions@1.0.0
  common-review-rubrics@1.0.0
  prd-writing@1.0.0
  ...

$ hx hub golden
package	api-conventions@1.0.0
package	prd-writing@1.0.0
bundle	api-service@1.0.0
bundle	frontend-2c@1.0.0

$ hx hub search prd --hub ./harness-hub
package	prd-writing@1.0.0	guide.skill

$ hx hub search --hub ./harness-hub --category blueprint
blueprint	enterprise-delivery@1.0.0	harness.blueprint	enterprise-delivery

$ hx hub search --hub ./harness-hub --index
wrote ./harness-hub/index.json
```

### 2. 业务仓库：从 Hub Bundle 一键初始化

```console
$ cd member-api
$ hx init --from-hub api-service@1.0.0 --hub ../harness-hub --adapter cursor
Initialized from hub api-service@1.0.0 → /work/member-api/harnessX
  + constitution.md
  + config.yaml
  + harness.yaml
  + hub bundle api-service@1.0.0
  ...

Next steps:
  1. Edit harnessX/constitution.md — write your project principles
  2. Sync adapters:             hx adapter sync
  ...
```

`config.yaml` 自动写入 `hub: ../harness-hub`。`harness.lock` 已生成，锁定解析到的资产哈希。

验证拓扑能力（含 v0.4 的 `integration-smoke` 槽位）：

```console
$ hx bundle list --hub ../harness-hub
api-service@1.0.0	(hub)
frontend-2c@1.0.0	(hub)

$ hx hooks install && hx ci init && hx adapter sync
cursor (Tier 1): 42 file(s)
```

### 3. 可选：企业交付蓝图

若项目走完整 enterprise 路径（需求调研 → PRD → 原型 → UAT），改用蓝图初始化：

```console
$ hx init --from-hub enterprise-delivery@1.0.0 --hub ../harness-hub
```

蓝图 `enterprise-delivery` 会：

- 将 `config.yaml` 的 profile 设为 `enterprise`
- 安装 `prd-writing`、`prototype-wireframe`、`uat-checklist` 等 Hub 依赖
- 写入 `harnessX/blueprint.yaml`

```yaml
# harnessX/blueprint.yaml（节选）
name: enterprise-delivery
extends: enterprise
hub_deps:
  - requirements-research-outline@1.0.0
  - prd-writing@1.0.0
  - prototype-wireframe@1.0.0
  - uat-checklist@1.0.0
```

### 4. Hub 升级与本地定制的三方合并

两个月后平台组发布 `api-conventions@1.1.0`。周工例行同步：

```console
$ hx hub sync --hub ../harness-hub
api-conventions	installed 1.0.0	latest 1.1.0	update-available
```

若周工曾在 `.hub-cache/api-conventions/` 加过本地条目，状态会变为 `update-and-local-changes`。应用合并：

```console
$ hx hub sync --hub ../harness-hub --apply
api-conventions	updated → 1.1.0
run hx lock write to refresh harness.lock

$ hx lock write && hx lock verify
harness.lock verified
```

冲突时（同一文件双方均改）：

```console
$ hx hub sync --hub ../harness-hub --apply
api-conventions	conflict	merge conflicts — use --force to apply with conflict markers

$ hx hub sync --hub ../harness-hub --apply --force
api-conventions	conflict → 1.1.0 conflicts: SKILL.md
```

`--force` 在冲突处写入 `<<<<<<< local` / `>>>>>>> upstream` 标记，人工裁决后 `hx lock write`。

### 5. Steering → Hub 闭环发布

周工在 member-api 沉淀了一条经过验证的 Skill，走闭环发布：

```console
$ hx asset backfill harnessX/assets/guides/idempotency-keys
metrics: runs=48, failures=1

$ hx steer publish harnessX/assets/guides/idempotency-keys \
    --hub ../harness-hub --by zhou.dev
eval: PASS (5 checks)
published to ../harness-hub/packages/idempotency-keys/0.1.0 (local trial, hub review pending)

$ hx hub approve idempotency-keys@0.1.0 --hub ../harness-hub --reviewer zhao.platform
```

发布前 `hx steer publish` 自动执行 `hub eval`（schema、注入扫描、内容完整性）；`draft` 资产会先提升到 `trial`。

### 6. Codex CLI 与 Tier 补偿

团队有人用 Codex 而非 Cursor：

```console
$ hx adapter sync --targets codex,generic
codex (Tier 2): 12 file(s)
generic (Tier 2): 8 file(s)

$ cat .harnessx-adapter-tier
2
```

Tier 2 下 `hx gate check` 会自动追加 `typecheck`、`lint`、`spec-validate` 到 gate 套件，并将部分 warn 升级为 block。终端挂机：

```console
$ hx apply add-member-tier --runner 'codex exec --prompt "$HX_TASK_TITLE"'
```

## 关键机制解析

| 能力 | 说明 |
| --- | --- |
| `init --from-hub` | Bundle/蓝图/单包三种 Hub 引用，统一初始化 + lock |
| `blueprint.yaml` | 可复制的交付路径（profile + hub_deps + 阶段资产） |
| `hub sync --apply` | 基线 / 本地 override / 上游 三方合并 |
| Tier → Gate 补偿 | 弱 IDE 提高 L3 检查强度，弥补缺少 Hooks 的缺口 |
| `steer publish` | metrics → eval → hub promote，证据链内置 |
