## 背景

安全组维护一个内部敏感信息扫描器（Python 写的 `secscan`）。他们希望：

1. 把 secscan 作为 Sensor 挂进 verification 套件，**输出结构化报告**；
2. 对 `config/` 与 `*.env.example` 做**保存即扫**；
3. 扫出问题时一键生成带上下文的修复会话。

## 操作步骤

### 1. 用 `check: shell` 接入扫描器

Shell Sensor 在仓库根执行命令：exit 0 = 通过；stdout 可含一行 JSON `SensorReport`。可用 `$HX_ROOT` / `$HX_CHANGE` / `$HX_OUTPUT` 等（见配置手册）。

```bash
#!/usr/bin/env bash
# harnessX/assets/sensors/secscan/check.sh
set -euo pipefail
secscan --json "${HX_ROOT:-.}" | ... # 转成 SensorReport JSON 并 exit
```

```yaml
sensors:
  - id: secscan
    kind: sensor.script
    execution: computational
    stage: dev
    task: verify
    check: shell
    run: "bash assets/sensors/secscan/check.sh"
    on_fail: block
    timeout_ms: 180000

suites:
  verification: [spec-validate, spec-trace, secscan]
```

**fail-closed**：崩溃、超时、非法 JSON → error，gate 阻断。

### 1b. 三种配置（inline / shell / rules）

```yaml
- id: spec-validate
  check: inline
  expr: "spec.ears_ok == true"
  source: assets/sensors/spec-validate

- id: no-todo-prd
  check: inline
  expr: "rules.list_ok"
  source: assets/sensors/example-rule
  on_fail: warn
```

### 2. 高危路径保存即扫（file-save）

```yaml
  - id: secscan-hot
    kind: sensor.script
    execution: computational
    trigger: file-save
    scope: ["config/**", "**/*.env.example", "deploy/**"]
    check: shell
    run: "bash assets/sensors/secscan/check.sh"
    on_fail: block
```

```console
$ hx watch
```

### 3. 一键修复：hx fix

```console
$ hx fix --change fee-recalc --sensor secscan
```

### 4. 误报治理：waiver，而不是关掉规则

```console
$ hx waiver add fee-recalc --target secscan \
    --reason "测试种子数据身份证号，已与安全确认" \
    --requested-by li.dev --approved-by security.zhao \
    --expires 2026-10-01T00:00:00Z
```

## 关键机制

- **三种 check**：`inline`（谓词 / `handler.*`）、`shell`（外部工具）、`rules`（LLM/启发式）
- **三种触发**：阶段门禁、file-save、schedule
- **结构化报告驱动修复回环**：`findings` + `fix_command`
