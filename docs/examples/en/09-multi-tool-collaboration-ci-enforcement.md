## Background

Settlement service (settlement-service) team tool preferences differ: Li uses Cursor, Zhou Claude Code, Zheng Trae (likes planner/executor dual agents), contractor uses Qoder for long Quest tasks. Some also use Copilot occasionally.

Pain: cannot maintain separate rule files per tool — four copies destined to drift. HarnessX answer: **`harnessX/assets/` single source, adapter compiles to each tool's native format**; generated files are "build artifacts", no hand edits.

## Steps

### 1. One compile, each tool takes what it needs

```console
$ hx adapter sync
cursor (Tier 1): ... file(s)
trae (Tier 1): ... file(s)
qoder (Tier 1): ... file(s)
claude (Tier 1): ... file(s)
generic (Tier 2): ... file(s)
```

File counts depend on the profile tasks and bound assets; the sample count is not a stable contract.

Same commands and rules land in each tool's native locations. Command **prompt bodies** come from single-source `harnessX/assets/commands/<task>.md` (`guide.command` type): each slash command is a thin task checklist; adapter sync appends the Context Pack, bound Skills/Templates, suite Sensors, and Gate reminder automatically. Running `/hx-dev-propose` in-tool gives the agent that task checklist and generated context.

| Tool | Generated artifacts | Notes |
| --- | --- | --- |
| Cursor | `.cursor/commands/hx-*.md`, `.cursor/skills/*/SKILL.md`, `.cursor/rules/harnessx.mdc`, `.cursor/hooks.json` | Command files carry thin task checklists plus auto-enriched context/sensor appendices; hooks run `hx gate hook-check` before commit prompts, `hx fixture verify` after editing fixtures/meta.yaml |
| Claude Code | `CLAUDE.md`, `.claude/commands/hx-*.md`, `.claude/settings.json` | Same command files; settings **deny editing meta.yaml / fixtures / lock files** (L2 enforcement) |
| Trae | `.trae/rules/project_rules.md`, `.trae/agents.yaml` | agents.yaml defines `hx-planner` (propose/design/plan) and `hx-executor` (apply/verify), with separate command whitelists |
| Qoder | `.qoder/rules/harnessx.md`, `.qoder/skills/*.md`, `.qoder/commands/hx-*.md`, `.qoder/mcp.json` | Plus Quest export (step 3) |
| Other (Copilot etc.) | `AGENTS.md` | Tier 2 fallback: rules + command docs — at least "know the rules" |

Change command workflows only in `harnessX/assets/commands/` single source, re-run `hx adapter sync` → all tools updated; same prompts also enter Context Pack (`hx guide pack`) — CLI and tools see the same playbook.

Same change, same phase, four people's entry points:

```text
Li (Cursor)      ▸ Agent dialog /hx-dev-propose fee-recalc
                      (slash completion from .cursor/commands/; Skills auto-mount;
                        hooks.json runs hx gate hook-check before commit prompts)
Zhou (Claude Code) ▸ Terminal claude, session /hx-dev-propose fee-recalc
                      (commands from .claude/commands/; settings.json denies
                        meta.yaml/fixtures/lock edits — harder than discipline)
Zheng (Trae)        ▸ Select hx-planner agent for planning tasks
                      (agents.yaml limits to propose/design/plan hx commands;
                        implementation must switch to hx-executor, whitelist changes)
Contractor (Qoder)  ▸ Open .qoder/quests/fee-recalc.md to start Quest (step 3)
```

Four paths, one change workspace, same gates — tools are steering wheels, harness is the same reins.

### 2. Hand-edit detection: compile output is not an edit surface

Zhou shortcuts by adding a team convention directly in `CLAUDE.md`. CI drift check catches it:

```console
$ hx adapter drift
DRIFT CLAUDE.md: manually-edited
```

Each generated file header has `GENERATED by harnessx adapter v1.0.0 ... (hash:...)`, drift recalculates body hash. Correct path: write convention in `harnessX/assets/guides/` Skill (or constitution), then `hx adapter sync` — **one edit, five targets**.

### 3. Qoder long tasks: Quest export and worktree isolation

Contractor takes planned change (`fee-recalc`, 7 tasks). Pack delta spec + tasks into Quest input:

```console
$ hx adapter quest fee-recalc
wrote .qoder/quests/fee-recalc.md
```

Quest file includes full spec, task list (with Requirement labels), environment guidance (`git worktree add ../quest-fee-recalc` isolated run, `hx verify fee-recalc` inside worktree before merge). **Worktree isolation + verify gate** means: however Qoder works inside, only path to mainline is green verification suite.

Since v0.2, HarnessX provides unified worktree CLI (`hx runtime worktree create`), **best-of-N fan-out** (`hx apply --fan-out N`), and diff annotation import (`hx review import`) — full walkthrough in [scenario 13](13-v0.2-orchestration-parallel-delivery.md).

### 4. Bypass drill: `--no-verify` does not go far

Zheng rushes home, local fast suite red, bypasses hooks:

```console
$ git commit -m "wip: fee recalc" --no-verify
$ git push --no-verify
```

Push succeeds (L2 bypassable). PR `harness-verify` workflow (L3) replays all checks:

```console
Replay gates for active changes
  fee-recalc [apply]: BLOCKED
BLOCKER  lint: exit 1
GATE BLOCKED (apply)
Error: Process completed with exit code 1.
```

Branch protection requires status check — PR cannot merge. **Local hooks save developer time (early feedback); CI replay is the enforcement source.**

### 5. Stealthier bypass: hand-edit meta.yaml

Agent (or human) notices gate state in `meta.yaml`, tries changing `task: apply` to `task: verify`. CI next step blocks this:

```console
Verify meta.yaml integrity (FR-050)
  fee-recalc: TAMPERED
  - meta.yaml contentHash mismatch — file was edited outside the hx CLI
Error: Process completed with exit code 1.
```

Only hx CLI writes meta.yaml with contentHash; gate records bind sensor log prefix hashes — changing status or history logs, both blocked. Claude permissions deny + Cursor hooks (L2 early intercept) add layers against "ledger tampering".

## Key mechanisms

- **Capability declaration not vendor hardcoding**: `hx adapter targets` shows each tool's declared capabilities (commands/skills/rules/hooks/agents/mcp/permissions), Tier from capabilities. New tool = one emitter + capability declaration, core unchanged.
- **"Single source + compile + drift detection" triad**: "don't hand-edit build artifacts" applied to prompt assets. Without drift detection, single-source promise collapses in three weeks.
- **Unified convergence under heterogeneous tools**: agent capabilities vary (some support hooks, some not), but all paths converge on same CLI gates — `hx verify` green is the bar. Tools interchangeable; convergence point is unique.
