# Scenario 09: One Repo, Four AI Tools, and CI That Cannot Be Bypassed

## Background

Settlement service (settlement-service) team tool preferences differ: Li uses Cursor, Zhou Claude Code, Zheng Trae (likes planner/executor dual agents), contractor uses Qoder for long Quest tasks. Some also use Copilot occasionally.

Pain: cannot maintain separate rule files per tool â€” four copies destined to drift. HarnessX answer: **`harnessX/assets/` single source, adapter compiles to each tool's native format**; generated files are "build artifacts", no hand edits.

## Steps

### 1. One compile, each tool takes what it needs

```console

trae (Tier 1): 2 file(s)
qoder (Tier 1): 4 file(s)

```

Same commands and rules land in each tool's native locations. Command **prompt bodies** come from single-source `harnessX/assets/commands/<phase>.md` (`guide.command` type): each slash command is not a one-line "run CLI" bridge but the **full phase workflow prompt** â€” steps, Guardrails, done criteria. Running `/hx-propose` in-tool gives the agent that phase playbook.

### 2. Hand-edit detection: compile output is not an edit surface

Zhou shortcuts by adding a team convention directly in `CLAUDE.md`. CI drift check catches it:

```console
$ hx adapter drift
DRIFT CLAUDE.md: manually-edited
```

Each generated file header has `GENERATED by harnessx adapter v1.0.0 ... (hash:...)`, drift recalculates body hash. Correct path: write convention in `harnessX/assets/guides/` Skill (or constitution), then `hx adapter sync` â€” **one edit, five targets**.

### 3. Qoder long tasks: Quest export and worktree isolation

Contractor takes planned change (`fee-recalc`, 7 tasks). Pack delta spec + tasks into Quest input:

```console
$ hx adapter quest fee-recalc
wrote .qoder/quests/fee-recalc.md
```

Quest file includes full spec, task list (with Requirement labels), environment guidance (`git worktree add ../quest-fee-recalc` isolated run, `hx verify fee-recalc` inside worktree before merge). **Worktree isolation + verify gate** means: however Qoder works inside, only path to mainline is green verification suite.

### 4. Bypass drill: `--no-verify` does not go far

Zheng rushes home, local fast suite red, bypasses hooks:

```console
$ git commit -m "wip: fee recalc" --no-verify
$ git push --no-verify
```

Push succeeds (L2 bypassable). PR `harness-verify` workflow (L3) replays all checks:

```console
Replay gates for active changes
  fee-recalc [apply]: BLOCKED
BLOCKER  lint: exit 1
GATE BLOCKED (apply)
Error: Process completed with exit code 1.
```

Branch protection requires status check â€” PR cannot merge. **Local hooks save developer time (early feedback); CI replay is the enforcement source.**

### 5. Stealthier bypass: hand-edit meta.yaml

Agent (or human) notices gate state in `meta.yaml`, tries changing `status: implementing` to `status: verified`. CI next step blocks this:

```console
Verify meta.yaml integrity (FR-050)
  fee-recalc: TAMPERED
  - meta.yaml contentHash mismatch â€” file was edited outside the hx CLI
Error: Process completed with exit code 1.
```

Only hx CLI writes meta.yaml with contentHash; gate records bind sensor log prefix hashes â€” changing status or history logs, both blocked. Claude permissions deny + Cursor hooks (L2 early intercept) add layers against "ledger tampering".

## Key mechanisms

- **Capability declaration not vendor hardcoding**: `hx adapter targets` shows each tool's declared capabilities (commands/skills/rules/hooks/agents/mcp/permissions), Tier from capabilities. New tool = one emitter + capability declaration, core unchanged.
- **"Single source + compile + drift detection" triad**: "don't hand-edit build artifacts" applied to prompt assets. Without drift detection, single-source promise collapses in three weeks.
- **Unified convergence under heterogeneous tools**: agent capabilities vary (some support hooks, some not), but all paths converge on same CLI gates â€” `hx verify` green is the bar. Tools interchangeable; convergence point is unique.
