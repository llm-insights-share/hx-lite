## Steps

### 1. Platform team: seed Hub and search assets

```console
$ hx hub seed ./harness-hub

  prd-writing@1.0.0
  ...

$ hx hub search prd --hub ./harness-hub

$ hx hub search --hub ./harness-hub --category blueprint

```

### 2. Application repo: one-shot init from Hub bundle

```console
$ cd member-api
$ hx init --from-hub api-service@1.0.0 --hub ../harness-hub --adapter cursor
Initialized from hub api-service@1.0.0 â†’ /work/member-api/harnessX
  + hub bundle api-service@1.0.0
  ...

$ hx bundle list --hub ../harness-hub
api-service@1.0.0	(hub)
frontend-2c@1.0.0	(hub)

$ hx hooks install && hx ci init && hx adapter sync
```

`--hub` can also point directly to a GitHub private repository (SSH recommended):

```console
$ hx init --from-hub api-service@1.0.0 --hub git@github.com:your-org/hx-hub.git --adapter cursor
$ hx hub search prd --hub git@github.com:your-org/hx-hub.git
```

On first use, HarnessX clones to local cache `harnessX/.hub-remotes/<hash>/repo`; later commands auto-fetch updates. Ensure `ssh -T git@github.com` works and your key has repo access.

`config.yaml` records `hub: ../harness-hub`; `harness.lock` pins resolved asset hashes.
