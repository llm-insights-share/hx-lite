# 场景 17：v0.4 平台治理 — 原型/UAT 门禁、漂移检测与组织看板

## 背景

SaaS 团队在 **enterprise** profile 下交付 **会员徽章** 功能（延续场景 15），v0.4 补齐了三块平台能力：

1. **阶段 Sensor**：`prototype-complete`（设计线框）、`uat-complete`（用户验收清单）
2. **统一漂移**：`drift` sensor（spec↔code、design↔code、adapter 手改）
3. **组织治理**：`hx steer coverage --aggregate` 跨仓指标、`hx view` 阶段漏斗与资产效果

角色：**林设计**（原型/HLD）、**陈产品**（UAT 签收）、**赵老师**（平台组看板）。

## 操作步骤

### 1. Design — 原型门禁 `prototype-complete`

UI 在范围内时，enterprise 的 `design-enterprise` 套件含 `prototype-complete`：

```console
$ hx gate check member-badge --phase design
BLOCKED: prototype-complete: UI in scope but design/ui/pages.md missing
```

林设计按 `prototype-wireframe` Skill 补充线框：

```text
Cursor ▸ /hx-design member-badge
         参考 Hub 资产 prototype-wireframe，填写 design/ui/pages.md
```

```markdown
# design/ui/pages.md（节选）
| Page | Route | Shell | Purpose |
|------|-------|-------|---------|
| MemberBadge | /member/badge | PortalShell | 展示用户徽章列表 |
```

```console
$ hx gate check member-badge --phase design
GATE PASS (design)
```

### 2. Verify — UAT 门禁 `uat-complete`

verify 前，陈产品从 Hub 模版复制验收清单：

```console
$ hx hub add uat-checklist@1.0.0 --hub ../harness-hub
installed uat-checklist@1.0.0 → harnessX/.hub-cache/uat-checklist
```

在 change 目录创建 `uat-checklist.md`（或 `requirements/uat-checklist.md`），填写场景走查表：

```markdown
## Scenario Walkthrough
| Scenario ID | Steps | Expected | Pass | Tester | Date |
|-------------|-------|----------|------|--------|------|
| s-badge-list | 打开 /member/badge | 显示已解锁徽章 | [x] | chen.pm | 2026-07-07 |

## Sign-off
- [x] Product owner approves behaviour vs PRD
**Approver:** chen.pm  **Date:** 2026-07-07
```

```console
$ hx gate check member-badge --phase verify
GATE PASS (verify)   # uat-complete 通过
```

### 3. 统一漂移检测 `drift` sensor

`verification` 与 `verification-enterprise` 套件已包含 `drift`，聚合三类信号：

| 规则 | 来源 |
| --- | --- |
| `spec-code-drift` | 归档 spec 与测试引用不一致（同 `hx sync`） |
| `design-code-drift` | `delivery-trace.yaml` 的 `code_hints` 未落地 |
| `adapter-drift` | `.cursor/rules/harnessx.mdc` 等被手改 |

单独巡检：

```console
$ hx sync
[scenario-without-test] Scenario: badge-empty-state
  → Spec declares "badge-empty-state" but no test references it...

$ hx gate check member-badge --phase verify
warning  drift: [scenario-without-test] Scenario: badge-empty-state — ...
```

修复测试或回写 spec 后重新 verify。手改 adapter 产出物时：

```console
$ echo "# manual tweak" >> .cursor/rules/harnessx.mdc
$ hx adapter drift
DRIFT .cursor/rules/harnessx.mdc: manually-edited

$ hx adapter sync    # 恢复单源编译产物
```

### 4. 集成测试槽位 `integration-smoke`

api-service 拓扑的 verification 套件含 `integration-smoke`。在 `package.json` 声明：

```json
"scripts": {
  "test:integration": "vitest run --config vitest.integration.config.ts"
}
```

```console
$ hx gate check member-badge --phase verify
# integration-smoke 执行 npm run test:integration
```

无该脚本时 sensor **跳过**（info），不阻断 gate。

### 5. 组织级 Harness Coverage 聚合

赵老师维护 12 个微服务目录 `~/corp/services/*`，每月汇总质量指标：

```console
$ hx steer coverage --aggregate ~/corp/services
aggregated 12 repo(s) → harnessX/coverage-index.json
avg first-attempt pass: 73.2%
recurrent patterns: 4, uncovered: 1
```

在任意已接入仓库查看本仓报告：

```console
$ hx steer coverage
sensor runs: 842
first-attempt pass rate: 71.0%
recurrent patterns (>=3): 2, uncovered: 0
```

### 6. 交付仪表盘 `hx view`（v0.4）

```console
$ hx view --out harnessx-dashboard.html
wrote /work/member-api/harnessx-dashboard.html
```

仪表盘包含：

- **指标卡片**：Sensor runs、首次通过率、复发/未覆盖模式
- **Phase funnel**：各阶段活跃 change 数量
- **Changes 表**：任务进度、场景覆盖、最近 gate
- **Asset effectiveness**：按 runs 排序的资产效果表

```console
$ hx status
member-badge	implementing	enterprise	tasks 3/4	scenarios 2/3
loyalty-coupon	proposed	standard	tasks 0/0	scenarios 0/2
```

## 关键机制解析

| Sensor / 命令 | 阶段 | 作用 |
| --- | --- | --- |
| `prototype-complete` | design | UI 在 scope 时强制 `design/ui/pages.md` |
| `uat-complete` | verify | 强制 UAT 清单与签收区 |
| `drift` | verify | 统一 spec/design/adapter 漂移 |
| `integration-smoke` | verify | 有脚本时跑集成测试 |
| `hx steer coverage --aggregate` | — | 多仓 Harness Coverage 写入 `coverage-index.json` |
| `hx view` | — | 阶段漏斗 + 资产效果可视化 |
