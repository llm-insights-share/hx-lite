# 场景 15：企业级需求分析 → 概要/详细设计 → AI 编码交接（enterprise profile）

## 背景

某 SaaS 团队采用 **enterprise** profile，要求每次 change 具备完整制品链：

1. **需求分析**：`requirements/`（PRD 蒸馏）
2. **概要设计（HLD）**：`design/overview.md`
3. **详细设计（LLD）**：`design/ui/`、`design/api/`、`design/data/`
4. **行为规格**：delta spec + `delivery-trace.yaml`
5. **编码交接**：`tasks.md` 含 `@design=` / `@files=`，`hx guide task-pack` 按任务注入上下文

角色：**陈产品**（PRD/propose）、**林设计**（HLD/LLD）、**赵前端**（apply）。

## 0. 初始化

```console
$ hx init --bundle frontend-dashboard
$ hx change create member-badge --domains member --profile enterprise
```

## 1. Propose — 需求分析 + 提案

```text
Cursor ▸ /hx-propose member-badge
         PRD：@docs/prd/member-badge.md
```

```console
$ hx propose member-badge --title "会员徽章展示"
wrote harnessX/changes/member-badge/proposal.md
wrote harnessX/changes/member-badge/requirements/prd-summary.md
wrote harnessX/changes/member-badge/traces/delivery-trace.yaml
```

填写 `requirements/user-stories.md`，在 `proposal.md` 填写 **PRD Reference**。

```console
$ hx gate check member-badge --phase propose
GATE PASS (propose)
```

## 2. Design — HLD + LLD

```console
$ hx design member-badge
wrote harnessX/changes/member-badge/design/overview.md
```

林设计补充：

- `design/overview.md` — API Surface、ADR、Design Tokens 表
- `design/ui/pages.md` — 徽章页路由与 `PortalShell`
- `design/ui/components/member-badge.md` — 组件 props、状态、a11y

```console
$ hx gate check member-badge --phase design
GATE PASS (design)
```

## 3. Spec → 批准 → Plan

陈产品 `/hx-spec` 定稿 delta spec；终端批准：

```console
$ hx gate approve member-badge --gate spec --approver chen.pm
$ hx plan member-badge
wrote harnessX/changes/member-badge/tasks.md (4 tasks)
```

`tasks.md` 示例：

```markdown
- [ ] 01a [test] (member / Requirement: 会员徽章展示) ... @design=design/overview.md @files=tests/member/badge.test.ts
- [ ] 01b [impl] (member / Requirement: 会员徽章展示) ... @design=design/ui/components/member-badge.md @depends=01a
```

## 4. Apply — Task Pack 驱动编码

```console
$ hx guide task-pack member-badge 01b
wrote harnessX/changes/member-badge/tasks/01b-pack.md (6 sections)
```

赵前端在 Cursor `/hx-apply` 中按任务实现；或：

```console
$ hx apply member-badge --runner '<agent>'
# 每任务设置 HX_TASK_PACK=.../01b-pack.md
```

## 5. Verify

```console
$ hx verify member-badge
$ hx gate check member-badge --phase verify
```

enterprise 的 `verification-enterprise` 套件含 `design-drift`：检查 `delivery-trace.yaml` 中的 `code_hints` 是否落地。

## 关键机制

| 能力 | 体现 |
| --- | --- |
| `requirements/` | PRD 蒸馏与 propose 门禁 |
| `design/overview.md` + LLD 目录 | HLD/LLD 分包 |
| `delivery-trace.yaml` | AC → Requirement → design → tasks → code |
| `@design=` / `@files=` | plan 自动生成交接注解 |
| `hx guide task-pack` | apply 阶段按任务裁剪 Context Pack |
| `enterprise` profile | propose/design/plan/verify 增强 Sensor 套件 |
