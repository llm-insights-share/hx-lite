# 场景 15：企业级需求分析 → 概要/详细设计 → AI 编码交接（enterprise profile）
| | |
| --- | --- |
| **旅程** | 企业交付 |
| **适用角色** | 企业 BA/架构 |
| **前置** | 场景 01；req/arch 阶段 见 [场景 19](19-组织级PRD与架构设计.md) |
| **关联** | 见 [场景选择指南](00-场景选择指南.md) |

## 背景

某 SaaS 团队采用 **enterprise** profile。完整企业旅程：**先**完成场景 19（`docs/prd/` + `docs/architecture/`），**再**为本 change 建立制品链：

1. **需求分析**：`requirements/`（PRD 蒸馏）
2. **概要设计（HLD）**：`design/overview.md`
3. **详细设计（LLD）**：`design/ui/`、`design/api/`、`design/data/`
4. **行为规格**：delta spec + `delivery-trace.yaml`
5. **编码交接**：`tasks.md` 含 `@design=` / `@files=`，`hx guide task-pack` 按任务注入上下文

角色：**陈产品**（PRD/propose）、**林设计**（HLD/LLD）、**赵前端**（apply）。

## 0. 初始化与 change 创建

```console
$ hx init --bundle frontend-dashboard
$ hx adapter sync
```

**前置**：场景 19 已完成 `docs/prd/member-badge.md` 与 `docs/architecture/modules/member/lld.md` 且已人工批准。

```console
$ hx change create member-badge \
    --domains member \
    --profile enterprise \
    --prd member-badge \
    --arch-modules member
```

`hx guide pack member-badge --stage dev --task propose` 输出的 Context Pack 已包含 **Org PRD** 与 **Org module LLD**（M20 自动注入）。

## 1. dev:propose — 需求分析 + 提案

```text
Cursor ▸ /hx-dev-propose member-badge
         PRD：@docs/prd/member-badge.md
```

```console
$ hx propose member-badge --title "会员徽章展示"
wrote harnessX/changes/member-badge/proposal.md
wrote harnessX/changes/member-badge/requirements/prd-summary.md
wrote harnessX/changes/member-badge/traces/delivery-trace.yaml
```

填写 `requirements/user-stories.md`，在 `proposal.md` 填写 **PRD Reference**。

```console
$ hx gate check member-badge --stage dev --task propose
GATE PASS (dev/propose)
```

## 2. dev:design — HLD + LLD

```console
$ hx design member-badge
wrote harnessX/changes/member-badge/design/overview.md
```

林设计阅读 Context Pack 中的 **Org architecture HLD** 与 **模块 LLD**，并补充 change 级设计：

- `design/overview.md` — API Surface、ADR、Design Tokens 表
- `design/ui/pages.md` — 徽章页路由与 `PortalShell`
- `design/ui/components/member-badge.md` — 组件 props、状态、a11y

```console
$ hx gate check member-badge --stage dev --task design
GATE PASS (dev/design)
```

enterprise design 套件含 **`arch-approved`**（全局 HLD 须已 `hx approve arch`）与 **`arch-change-align`**（touched domains 映射到模块 LLD）。

## 3. design-to-plan 批准 → dev:plan

陈产品在 `/hx-dev-design`（dev:design）中完成设计与 delta spec 定稿；终端批准后进 plan：

```console
$ hx gate approve member-badge --gate design-to-plan --approver chen.pm
$ hx plan member-badge
wrote harnessX/changes/member-badge/tasks.md (4 tasks)
```

`tasks.md` 示例：

```markdown
- [ ] 01a [test] (member / Requirement: 会员徽章展示) ... @design=design/overview.md @files=tests/member/badge.test.ts
- [ ] 01b [impl] (member / Requirement: 会员徽章展示) ... @design=design/ui/components/member-badge.md @depends=01a
```

## 4. Apply — Task Pack 驱动编码

```console
$ hx guide task-pack member-badge 01b
wrote harnessX/changes/member-badge/tasks/01b-pack.md (6 sections)
```

赵前端在 Cursor `/hx-dev-apply` 中按任务实现；或：

```console
$ hx apply member-badge --runner '<agent>'
# 每任务设置 HX_TASK_PACK=.../01b-pack.md
```

## 5. Verify

```console
$ hx verify member-badge
$ hx gate check member-badge --stage dev --task verify
```

enterprise 的 `verification-sdlc` 套件含 `design-drift` 与 `arch-drift`（未 promote 时 warn）。

## 6. 沉淀与归档

```console
$ hx arch promote member-badge --by lin.arch
promoted change "member-badge" → modules [member]

$ hx rebase check member-badge
$ hx archive member-badge
```

未执行 `hx arch promote` 时，enterprise **archive 会被阻断**（可用 `hx waiver add member-badge --target arch-drift --reason "temporary alignment pending arch promote" --requested-by zhao.fe --approved-by lin.arch` 豁免）。

## 关键机制

| 能力 | 体现 |
| --- | --- |
| `requirements/` | PRD 蒸馏与 propose 门禁 |
| `design/overview.md` + LLD 目录 | HLD/LLD 分包 |
| `delivery-trace.yaml` | AC → Requirement → design → tasks → code |
| `@design=` / `@files=` | plan 自动生成交接注解 |
| `hx guide task-pack` | apply 阶段按任务裁剪 Context Pack（含 org 模块 LLD 的 `@design=`） |
| `hx arch promote` | change design 结构化沉淀到 `docs/architecture/modules/*/lld.md` |
| `enterprise` profile | dev:propose/design/plan/verify 增强 Sensor 套件 |
