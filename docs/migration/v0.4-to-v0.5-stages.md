# v0.4 → v0.5 四阶段迁移指南

## 变更摘要

v0.5 引入 **req / arch / dev / test** 四阶段交付模型，作为可选的 `delivery_mode: stages`。默认仍为 `phases`，与 v0.4 完全兼容。

## 迁移步骤

1. **预览 meta 迁移**

```bash
hx migrate --dry-run
```

2. **写入 stage/task 字段**

```bash
hx migrate
```

3. **启用四阶段 gate（可选）**

```bash
hx migrate --delivery-mode stages
```

4. **验证**

```bash
hx stage status <change>
hx gate check <change> --stage dev --task propose
```

## meta.yaml 字段

| 新字段 | 说明 |
| --- | --- |
| `stage` | `req` \| `arch` \| `dev` \| `test` |
| `task` | 阶段内任务 id，如 `propose`、`test-case-design` |
| `stageProgress` | 各阶段完成度 |
| `taskHistory` | 任务 gate 历史 |

`status`（`proposed`、`designed` 等）在 `delivery_mode: phases` 下保留；`stages` 模式下仍同步更新。

## 旧 status → stage/task

| status | stage | task |
| --- | --- | --- |
| `explore` | req | requirements-research |
| `proposed` | dev | propose |
| `designed` | dev | design |
| `specified` | dev | design |
| `planned` | dev | plan |
| `test_designed` | test | test-case-design |
| `implementing` | dev | apply |
| `verified` | dev | verify |
| `archived` | dev | archive |

## 批准门变更

| v0.4 | v0.5（推荐） |
| --- | --- |
| `hx gate approve --gate spec` | `hx gate approve --gate design-to-plan`（`spec` 仍接受） |

## harness.yaml profile

新增字段（与 `phases` 并存）：

```yaml
profiles:
  standard:
    stages: [req, arch, dev, test]
    dev_tasks: [plan, propose, design, apply, verify, archive]
    suites:
      dev.verify: verification
```

## L1 契约

新增环境变量：

- `HX_STAGE`：`req` \| `arch` \| `dev` \| `test`
- `HX_TASK`：阶段内任务 id

`HX_PHASE` 保留，标记为 deprecated。

## 回滚

```yaml
# config.yaml
delivery_mode: phases
```

旧 CLI 命令（`hx prd`、`hx propose`、`hx gate check --phase`）继续可用。
