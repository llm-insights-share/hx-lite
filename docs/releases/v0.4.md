# HarnessX v0.4

**Tag:** `v0.4` · **npm version:** `0.4.0` · **Node:** ≥ 20

HarnessX v0.4 adds **native enterprise SDLC orchestration**: work orders, change requests, role-based permissions, a dedicated test-design phase, and a bug/retest loop — on top of the v0.3 artifact + gate control plane.

---

## Highlights

### Enterprise SDLC profile (`enterprise-sdlc`)

New workflow profile with an extra **test-design** phase:

```
explore → propose → design → spec → plan → test-design → apply → verify → archive
```

Initialize:

```bash
hx init --from-hub enterprise-sdlc@1.0.0 --hub <path>
```

Walkthrough: [Scenario 20](../examples/20-企业SDLC工单全流程.md)

### Work orders (`hx wo`)

Ticket-driven handoffs between roles:

| Type | Flow |
| --- | --- |
| `req-review` | Product → Tech manager approves PRD |
| `arch-review` | Architect → Tech manager; spawns `lld-design` per module |
| `req-change` / `arch-change` | Change request review |
| `test-case-review` | QA → Tech manager before apply |
| `bug-fix` / `retest` | Bug fix and retest loop |

Commands: `hx wo create/submit/approve/reject/done/inbox/extract`

Storage: `harnessX/workorders/WO-*.yaml`

### Change requests (`hx cr`)

Structured requirement/design changes with `add` / `modify` / `delete`:

```bash
hx cr create --kind requirement-change --action modify --prd <slug> ...
hx cr submit CR-00001 --by pm.chen
hx wo approve WO-00002 --by tm.zhang   # applies CR, invalidates stale approvals
```

### Roles (`harnessX/roles.yaml`)

Maps members to roles (`tech-manager`, `product-manager`, `architect`, `developer`, `tester`) with permission checks under `enterprise-sdlc`.

### Test cases & bugs

- `hx test-cases init/check/submit <change>` — test-design phase
- `hx bug create/list/fix/close <change>` — bug tracking with work orders
- Sensors: `test-cases-complete`, `test-cases-approved`, `bugs-closed`

### Module LLD approval (T-837)

```bash
hx approve arch-lld <module> --approver <name>
```

Sensor `arch-lld-approved` enforces per-module LLD sign-off on `enterprise-sdlc`.

### Pre-phase shortcuts

- `hx prd submit <slug> --by <name>` — creates `req-review` work order
- `hx arch submit --by <name> [--change <id>]` — creates `arch-review` work order

---

## Upgrade from v0.3

- Existing `enterprise` profile is unchanged; opt in via `profile: enterprise-sdlc` in `config.yaml` or Hub blueprint.
- `hx init` now scaffolds `harnessX/roles.yaml`, `workorders/`, and `change-requests/`.
- Phase count increases from 8 to 9 (`test_designed` state); update any custom tooling that hard-codes phase lists.
