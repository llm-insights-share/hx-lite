# HarnessX v0.3

**Tag:** `v0.3` · **npm version:** `0.3.0` · **Node:** ≥ 20

HarnessX v0.3 is the first tagged release of the **outer harness control plane** for AI coding agents. It delivers three major capability layers: **layered architecture** (minimal config + Hub supply chain), **enterprise delivery** (requirements → HLD/LLD → task-pack handoff), and **organization Pre-phase** (org-level PRD/architecture with approvals and structured promote).

---

## Highlights

### Organization Pre-phase (M18–M20)

Enterprise teams can maintain **org-level truth** in `docs/` before individual changes begin:

| Track | Path | Approval |
| --- | --- | --- |
| Org PRD | `docs/prd/<slug>.md` | `hx approve prd <slug> --approver <name>` |
| Global HLD | `docs/architecture/overview.md`, `registry.yaml` | `hx approve arch --approver <name>` |
| Module LLD | `docs/architecture/modules/<id>/lld.md` | validated via `hx arch lld check` |

**Cursor slash commands** (after `hx adapter sync`): `/hx-prd`, `/hx-arch`, `/hx-arch-lld`

**Dual-track model:** org artifacts are **auto-injected** into propose/design Context Packs; change design is **written back** to module LLD via `hx arch promote <change>` before enterprise archive.

**Gates:** `prd-complete`, `prd-approved`, `arch-approved`, `arch-change-align`, `arch-drift` (warn if not promoted); **archive blocks** without promote on enterprise profile.

Walkthrough: [Scenario 19 → 15](../examples/19-组织级PRD与架构设计.md) (zh) · [Scenario 19 → 15](../examples/en/19-org-prd-and-architecture.md) (en)

### Enterprise delivery

`enterprise` profile adds a full artifact chain per change:

- `requirements/` — PRD distillation from org PRD
- `design/overview.md` + `design/ui|api|data/` — change-level HLD/LLD
- `traces/delivery-trace.yaml` — end-to-end trace
- `tasks.md` with `@design=` / `@files=` — scoped coding handoff
- `hx guide task-pack <change> <taskId>` — per-task Context Pack for apply

Initialize: `hx init --from-hub enterprise-delivery@1.0.0 --hub <path>`

### Layered architecture & Hub (v0.3 core)

- **`imports:`** — minimal `harness.yaml` referencing topology bundles
- **`hx init --from-hub <bundle>@<ver>`** — bootstrap from org Hub
- **`hx hub sync --apply`** — three-way merge of upstream Hub assets
- **Blueprint install** — reusable delivery paths (`blueprint.yaml`)
- **Gate compensation** — Tier 2 adapters (Codex, generic) get stronger L3 checks; optional `warn → block` escalation
- **Hub lifecycle** — seed, promote, review, policy, search, eval, private GitHub remote

### Orchestration (v0.2, included)

- `hx apply --parallel N` — concurrent tasks in same `@group`
- `hx apply --fan-out N` — best-of-N isolated worktrees
- `hx review import` — diff line annotations → fix hints
- `hx notify`, `hx eval guides`, `hx change create --from-issue`

### Product library

Built-in topology bundles: `api-service`, `frontend-2c`, `frontend-dashboard`, `event-consumer`, `library-sdk`, `serverless-function`, `mobile-app`, `data-pipeline` (+ `*-cn` locales).

Fast sensors, mutation probe, design-template, `hx mcp` bridge for headless agents.

## New commands (Pre-phase)

```bash
# Org PRD
hx prd init <slug> && hx prd check <slug>
hx approve prd <slug> --approver <name>

# Global architecture
hx arch init && hx arch check
hx approve arch --approver <name>

# Module LLD
hx arch lld init <module> && hx arch lld check <module>

# Link org artifacts to a change
hx change create <id> --profile enterprise --prd <slug> --arch-modules <module>

# Promote change design back to org LLD (before archive)
hx arch promote <change> --by <architect>

# Pre-phase Context Packs
hx guide prd-pack <slug>
hx guide arch-pack
```

## Upgrade notes

- **First tagged release** — no prior `v0.2` tag; `package.json` moves from `0.2.0` → `0.3.0`.
- **Enterprise users:** complete org Pre-phase ([scenario 19](../examples/19-组织级PRD与架构设计.md)) before `enterprise` changes; use `--prd` and `--arch-modules` on `change create`.
- **Archive:** enterprise profile requires `hx arch promote` (or explicit `arch-drift` waiver).
- **Adapters:** run `hx adapter sync` after upgrade to get `/hx-prd`, `/hx-arch`, `/hx-arch-lld` slash commands.

## Quick start

```bash
npm install
node bin/hx.js init --bundle api-service
node bin/hx.js hooks install && node bin/hx.js ci init && node bin/hx.js adapter sync

# Standard change
node bin/hx.js change create my-feature --domains billing
# Cursor ▸ /hx-propose my-feature
node bin/hx.js gate approve my-feature --gate spec --approver alice
node bin/hx.js plan my-feature && node bin/hx.js apply my-feature --runner "<agent>"
node bin/hx.js verify my-feature && node bin/hx.js archive my-feature
```
