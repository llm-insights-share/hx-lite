# Release v0.6.0 — Four-stage model (no compatibility layer)

**Date:** 2026-07-10

v0.6 removes all legacy phase/compat dual paths. The product speaks only **req → arch → dev → test** with **stage + task** on every change.

## Breaking changes

- **Removed** `meta.status`, `PHASES`, `delivery_mode`, `hx migrate`, top-level `hx prd`, CLI `--phase`
- **Removed** legacy `gate.ts` nine-phase engine; unified `stageGate.ts`
- **Meta** requires `stage` + `task`; new changes start at `dev/propose`
- **Profiles** preferred shape: `stages` + `tasks: { req|arch|dev|test: [{ id, suite }] }` (legacy `*_tasks` + `suites: { "dev.x": name }` still normalize via `normalizeProfile`)
- **Slash commands**: `/hx-<stage>-<task>`; no `/hx-spec` / `cmd-spec` (delta finalize lives in `dev.design`)
- **Approvals** file: `docs/.stage-approvals.yaml` (replaces `.prephase-approvals.yaml`)
- **Design→plan gate** approval: `--gate design-to-plan` (replaces `spec`)
- **L1 contract** drops `HX_PHASE`; use `HX_STAGE` + `HX_TASK`
- **Work orders** scope: `req` | `arch` | `change` (replaces `prephase`)

## CLI command tree

```
hx req prd init|check|list|submit
hx req status
hx arch init|check|lld|promote|...
hx arch status
hx dev plan|propose|design|apply|verify|archive <change>
hx dev status <change>
hx test cases init|check|submit <change>
hx test status <change>
hx stage status [--stage req|arch|dev|test] [change]
hx gate check <change> --stage <s> --task <t>
hx gate advance <change>
hx gate approve --gate design-to-plan|prd|arch|...
hx guide pack <change> --stage <s> --task <t>
```

## Migration from v0.5

There is **no runtime migration**. Re-init or hand-edit `meta.yaml` to `stage`/`task`. Re-run `hx harness lint` after pulling bundle updates.
