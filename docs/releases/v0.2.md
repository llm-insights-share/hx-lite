# hx-lite v0.2

**Tag:** `v0.2` · **npm version:** `0.2.0` · **Node:** ≥ 20

hx-lite v0.2 clarifies **Profile → Stage → Task → Guide/Suite**, thins IDE command assets, and aligns scaffold/docs with the stage-task asset matrix.

---

## Highlights

### Preferred Profile shape

```yaml
profiles:
  standard:
    stages: [req, arch, dev, test]
    tasks:
      dev:
        - id: design
          suite: design-basic
        - id: apply
          suite: fast
```

- Prefer `profiles.*.tasks[].{id,suite}`
- Top-level `suites` are **named packs only** (e.g. `design-sdlc`); legacy `*_tasks` / `suites: { "dev.x": name }` still normalize via `normalizeProfile`
- Dead `*-enterprise` suite names removed from scaffold

### Thin commands + `/hx-<stage>-<task>`

- Command bodies are Input / Steps / Output / Guardrails / Done when
- Adapter appendix injects Context Pack, bound Skills/Templates, suite Sensors, and Gate reminders
- `spec.md` / `cmd-spec` merged into `dev.design` (`/hx-dev-design`); explore rewritten as org `requirements-research`

### Stage-task asset matrix

- Doc: [stage-task-assets.zh-CN.md](../stage-task-assets.zh-CN.md)
- New guides (requirements-analysis, tech-selection, database/interface/key-mechanisms, test authoring/execution, planning/archive checklists, …)
- Suites updated (`design-basic`, `verification-sdlc`, `archive-check`, `integration-smoke`, …)

### Docs & enterprise handbook

- [enterprise-delivery.zh-CN.md](../enterprise-delivery.zh-CN.md) role-oriented end-to-end
- Examples/glossary aligned to stage/task slash names and `design/overview.md`

Design notes: [design1.md](../../design1.md)
