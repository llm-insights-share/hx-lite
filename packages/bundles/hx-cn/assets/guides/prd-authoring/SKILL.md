# Skill: PRD authoring

When writing organization PRD at `docs/prd/<slug>.md`:

1. Every feature needs at least one user story (US-xxx) and acceptance criterion (AC-xxx).
2. Use Given / When / Then for acceptance criteria.
3. Separate In Scope vs Out of Scope explicitly.
4. NFR must be measurable (latency, availability, security).
5. Record review conclusion before propose phase.
